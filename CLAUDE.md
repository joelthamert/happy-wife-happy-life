# CLAUDE.md — Happy Wife Happy Life

## What this is
A partner-preference tracker with a liquid-glass UI. Users record what their partner loves (brands, musicians, foods, flowers, scents, sizes), track important dates with countdowns and calendar export, save gift ideas, run recurring reminders, and fill in a profile by flipping question flashcards ("Discover").

## Stack
- React 18 + Vite, TypeScript in `lib/`, `data/`, `hooks/`, `types.ts`, `theme.ts` and leaf components; remaining components/pages are `.jsx` (convert incrementally as touched).
- PWA via `vite-plugin-pwa` (manifest + workbox SW, icons in `public/` generated from the HWHL logo).
- All styling is inline style objects + the global CSS string in `src/theme.ts` (`GLOBAL_CSS`: keyframes, aurora, resets). No CSS framework. Do not introduce Tailwind without being asked.
- State: single `d: AppData` object owned by `src/hooks/useAppData.ts`, persisted through `src/storage.ts` (localStorage adapter mirroring `{ get → {value}, set }`).
- Storage key: `hwhl-v4`, wrapped in a `{ version, data }` envelope (`src/lib/migrations.ts`, CURRENT_VERSION = 7). To change the data shape: bump CURRENT_VERSION and add a migration step **in the same edit** — never silently drop user data. `load()` preserves unparseable payloads under `hwhl-v4-recovery` before falling back to defaults.
- Tests: Vitest (`npm test`) covers ICS, migrations, notifications, weekly rotation/streak, sale alerts, gift intelligence, map math, restaurants, couple sync helpers. `npm run typecheck` runs tsc.

## Design system rules (liquid glass)
- TWO THEMES via CSS variables: `html[data-theme="dark"|"light"]` selects the variable set defined in `GLOBAL_CSS` (`src/theme.ts`); dark is the designed default. Surface/text tokens (`T.bg/ink/body/faint/ghost`, `glass()`, `glassInput`, lines `--line*`, washes `--wash-*`, pastel accent text `--rose-text` etc.) are `var()` references — NEVER reintroduce literal white-alpha rgba or pastel hexes in components; use the vars. Accent hexes (rose/violet/champagne/mint) stay literal because alpha suffixes get appended (e.g. Liquid's glow). `settings.theme` persists (storage v8); `index.html` has a pre-paint boot script so there's no theme flash; App syncs `data-theme` + `<meta theme-color>`.
- Dark background `#0b0710` / light `#f4eef6` with the `Aurora` drifting light fields. Type: Instrument Serif italic for display, DM Sans for everything else. Eyebrows are 10.5px / 700 / 0.22em tracking / uppercase.
- Glass = translucent gradient fill + `blur(28px) saturate(170%)` + 1px `var(--glass-border)` + inset top highlight + the 1px specular streak. Use the `glass()` helper or `Liquid`.
- NEVER combine `transform-style: preserve-3d` with backdrop-filtered children — WebKit mispositions them (this detached the flashcard from its slot). FlashCard renders one face at a time with the `flipInY` animation instead.
- Mobile: `#hwhl-scroll` height/padding live in GLOBAL_CSS (`100dvh` + `env(safe-area-inset-bottom)`); the dock and sheets also pad for the home indicator.
- Accents: rose `#ff7a9c`, violet `#b07cff`, champagne `#e8c592`, mint `#7ce8b6` (mint = success/sync states only).
- Motion: easing `cubic-bezier(.2,1,.3,1)` for reveals, `.3,1.4,.4,1` for the dock spring. Ambient loops are 7s+. Respect `prefers-reduced-motion` (already wired — keep it working).
- Copy: sentence case, plain verbs ("Add to calendar", "Save date"), no exclamation marks in UI chrome.

## Conventions
- Artifact-portability: avoid `document.getElementById`/direct DOM where React state works; downloads must use the `downloadFile` pattern.
- Brand additions go through `addBrandTracked` so preferences and trackedBrands stay in sync (and removals must clean both).
- Flashcard answers route through `handleSaveAnswer` (splits on commas, auto-tracks brand answers).
- EVERY flashcard answer must feed a feature — `lib/linkage.test.ts` fails if a card has no consumer (pipeline mapTo, DATE_IDEA_CARD_IDS, or GIFT_INTEL_CARD_IDS). Add the consumer before adding a card. The audit lives in 'Discover question linkage.xlsx'.
- CROSS-LINKING is the product's core: musicians saved anywhere auto-resolve to followed artists via `lib/artistSync.ts` (debounced effect in `useAppData`; attempt log in `hwhl-artist-sync` so no-match names aren't re-searched and deliberate unfollows aren't re-followed); following an artist in Events backfills `preferences.musicians`; the Gifts concert suggestion (kind "concert" from `giftIntel`) deep-links to Events → Followed via App's `goEvents`. Preserve these loops when touching any of the three features.
- Run `npm run dev` to verify; there are no tests yet — if you add logic-heavy modules (ICS, storage migrations), add Vitest and cover them.
